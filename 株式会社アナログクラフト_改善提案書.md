# 株式会社アナログ・クラフト 業務改善提案書
## 部品・資材発注業務の自動化・標準化システム

---

## 1. 概要

本提案は、株式会社アナログ・クラフトにおける**部品・資材の発注業務**を対象とした、デジタル化・自動化システムの構築です。

現状の「手書きメモ → Excel転記 → 発注書作成」という二重入力プロセスを排除し、製造部と管理部がリアルタイムで情報を共有できる**統合発注管理システム**を提供します。

### 解決する課題
- ✅ 手書きメモからExcelへの転記作業の廃止
- ✅ 転記ミスによる発注トラブルの防止
- ✅ 在庫状況の可視化と適正在庫の維持
- ✅ 過去の発注データに基づく発注量の最適化提案
- ✅ 製造部と管理部の情報共有のリアルタイム化

### システムの特徴
- 📱 **クラウドベース**: Google Workspaceを活用し、どこからでもアクセス可能
- 🤖 **AI活用**: Gemini APIによる発注量の最適化提案
- 📊 **データの一元管理**: Google Sheetsによる情報の集約
- ⚡ **自動化**: Google Apps Script (GAS) による業務フローの自動化

---

## 2. 目的

### 業務目標
1. **発注業務の時間を50%削減**
   手書き + 転記作業（約2時間/日）→ 直接入力（約1時間/日）

2. **転記ミスをゼロにする**
   年間10件程度発生している転記ミスによる誤発注・納期遅延を完全に防止

3. **在庫の適正化**
   経験則による発注から、データに基づく発注へ移行し、過剰在庫を20%削減

4. **業務の標準化**
   属人的な判断を減らし、誰でも適切な発注判断ができる仕組みを構築

### 期待される効果（ROI）
- **時間削減**: 2時間/日 × 20営業日 = 40時間/月の業務時間削減
- **コスト削減**: 転記ミスによる損失（約10万円/年）+ 過剰在庫削減（約50万円/年）
- **品質向上**: 発注精度の向上により、製造計画の安定化

## 4. 入出力仕様

### 4.1 入力データ

#### 発注依頼情報
| 項目 | データ型 | 必須 | 説明 | バリデーション |
|------|---------|------|------|---------------|
| 資材名 | String | ✅ | 発注する資材の名称 | マスタから選択 |
| サプライヤー | String | ✅ | 発注先サプライヤー名 | マスタから選択 |
| 発注数量 | Number | ✅ | 発注する数量 | 1以上の整数 |
| 単位 | String | ✅ | 枚/本/L/kg等 | 資材マスタと連動 |
| 希望納期 | Date | ✅ | 納品希望日 | 今日以降の日付 |
| 用途/備考 | String | - | 発注理由や用途 | 500文字以内 |
| 依頼者 | String | ✅ | 発注依頼者名 | ログインユーザー |

#### 在庫情報
| 項目 | データ型 | 必須 | 説明 |
|------|---------|------|------|
| 資材ID | String | ✅ | 一意の資材識別子 |
| 現在在庫数 | Number | ✅ | 現時点の在庫数 |
| 安全在庫数 | Number | ✅ | 発注アラートの閾値 |
| 最終更新日時 | DateTime | ✅ | 在庫更新日時 |

### 4.2 出力データ

#### 発注書（PDF）
- 発注番号（自動採番: ORDER-YYYYMMDD-001）
- 発注日
- 発注先（サプライヤー名・住所・電話番号）
- 発注者（会社名・部署・担当者名）
- 資材詳細（資材名・数量・単価・小計）
- 合計金額
- 希望納期
- 備考

#### AI発注提案レポート
- 推奨発注量
- 算出根拠
  - 過去3ヶ月の平均使用量
  - 現在の在庫数
  - 今後の生産計画（Googleカレンダー連携）
  - リードタイム
- 発注タイミング

#### 在庫アラート通知（Gmail）
- 対象資材名
- 現在在庫数
- 安全在庫数
- 推奨アクション

## 11. まとめ

本提案は、株式会社アナログ・クラフトの発注業務における**転記作業の廃止**と**業務の標準化**を実現するものです。

### 提案のポイント
- ✅ **実現可能性が高い**: Google Workspaceという既存環境を活用
- ✅ **投資回収が早い**: 約5.5ヶ月で投資回収
- ✅ **段階的導入が可能**: プロトタイプから本番環境へスムーズに移行
- ✅ **AI活用**: 単なるデジタル化ではなく、データドリブンな意思決定を支援
- ✅ **拡張性**: 成功後、他の業務（顧客問い合わせ・設計図管理）へ展開可能

### 次のステップ
1. 経営層への提案・承認
2. プロジェクトチーム発足（製造部・管理部・IT担当）
3. プロトタイプ開発開始（まず2週間で動くものを作る）
4. 現場でのトライアル運用
5. 本番稼働

このシステムが成功すれば、**「デジタル化の第一歩」**として、他の業務改善への波及効果も期待できます。
